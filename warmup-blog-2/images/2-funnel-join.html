How a funnel step matches spans
Each step is a span-name filter. A partial match just lowers conversion — but a step that names a model no longer emitting spans matches zero, and then SigNoz crashes instead of reporting 0%.
n = 22 traces
Step 1 name = 'agent answer'
matches 22 traces ✓
Step 2 name = 'tool search_tool'
matches 22 traces ✓
Step 3 name = 'chat gpt-4o-mini'
matches 10 of 22 ⚠
12 traces switched to another model mid-experiment
step with zero matches ≠ 0% — the API crashes
Point a step at a model that stopped emitting spans (a fallback or version bump), it matches 0 spans, and the funnel API divides by that count without a guard:
HTTP 500
v3.Row.Data: unsupported value: NaN
